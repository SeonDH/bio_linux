# [번외] 응용 프로그램, 셸, 커널의 관계

리눅스를 처음 배우면 `응용 프로그램`, `셸`, `커널`이라는 단어가 자주 나온다.  
이 셋의 관계를 이해하면 이후에 나오는 `chmod`, `ps`, `cron`, `ssh`, `docker`, `slurm` 같은 명령어가 왜 그런 방식으로 동작하는지 훨씬 쉽게 이해할 수 있다.

## 1. 각각이 하는 일

### 응용 프로그램

응용 프로그램은 사용자가 직접 실행해서 쓰는 프로그램이다.

예를 들면 다음과 같다.

- 문서를 편집하는 프로그램
- 개발자가 코드를 작성하는 편집기
- 인터넷이나 네트워크 상태를 확인하는 도구
- 바이오 분석에 사용하는 스크립트나 실행 프로그램

응용 프로그램은 사용자가 원하는 일을 쉽게 하도록 만들어진 도구다.  
하지만 실제로 파일을 만들고, 지우고, 실행하고, 네트워크에 접속하고, 디스크에 기록하는 일은 응용 프로그램 혼자 하지 않는다. 그 아래에서 셸과 커널이 함께 동작한다.

### 셸(Shell)

셸은 사용자가 명령어를 입력하는 인터페이스다.  
리눅스에서 자주 보는 `bash`, `zsh`가 대표적인 셸이다.

셸의 역할은 단순히 글자를 받아 적는 것이 아니다.

- 사용자가 입력한 명령어를 해석한다.
- 명령어의 옵션과 인자를 분리한다.
- 필요한 경우 여러 명령어를 연결한다.
- 해석한 내용을 커널이 처리할 수 있는 요청으로 바꾼다.

즉, 셸은 사용자의 의도를 커널이 이해할 수 있는 형태로 바꿔주는 중간 계층이다.

### 커널(Kernel)

커널은 운영체제의 핵심이다.  
실제로 하드웨어와 시스템 자원을 관리한다.

커널이 담당하는 대표적인 작업은 다음과 같다.

- 메모리 관리
- 프로세스 관리
- 파일 시스템 관리
- 장치 관리
- CPU 스케줄링
- 디스크 I/O 처리
- 네트워크 처리

응용 프로그램이 파일을 읽거나, 프로세스를 실행하거나, 네트워크에 접속하는 일을 요청하면 커널이 그 일을 실제로 수행한다.

## 2. 동작 흐름

리눅스에서 사용자가 명령어를 입력했을 때의 흐름은 보통 다음과 같다.

1. 사용자가 터미널에서 명령어를 입력한다.
2. 터미널은 입력된 내용을 셸로 전달한다.
3. 셸은 명령어를 해석하고, 필요한 시스템 호출을 준비한다.
4. 커널은 요청을 받아 실제 작업을 수행한다.
5. 결과가 다시 셸과 터미널을 거쳐 사용자에게 표시된다.

예를 들어 `ls` 명령을 입력했다고 해보자.

- 사용자는 `ls`라고 입력한다.
- 셸은 `ls`가 디렉터리 목록을 보여 달라는 명령이라고 이해한다.
- 커널은 파일 시스템에 접근해서 해당 디렉터리 내용을 읽는다.
- 결과가 셸을 통해 화면에 출력된다.

이 구조를 알면, 나중에 나오는 대부분의 리눅스 명령어가 같은 흐름을 따른다는 것을 이해하기 쉽다.

## 3. 간단한 비유

이 셋의 관계를 아주 단순하게 비유하면 다음과 같다.

- 응용 프로그램은 **사용자가 직접 쓰는 도구**
- 셸은 **사용자와 운영체제를 이어 주는 창구**
- 커널은 **실제 작업을 처리하는 중심 엔진**

예를 들어 사용자가 파일을 복사하고 싶다고 하자.

1. 사용자는 셸에 `cp` 명령을 입력한다.
2. 셸은 `cp` 명령을 해석한다.
3. 커널은 원본 파일을 읽고, 새 위치에 복사한다.
4. 응용 프로그램이나 셸은 결과를 화면에 보여 준다.

사용자는 화면에서 단순히 한 줄의 명령만 보지만, 실제로는 이 뒤에서 여러 층이 함께 움직인다.

## 4. 왜 중요한가

이 관계를 이해하면 다음을 설명하기 쉬워진다.

- `chmod`가 왜 파일 권한을 바꾸는지
- `ps`가 왜 프로세스를 보여 주는지
- `cron`이 왜 정해진 시간에 작업을 실행하는지
- `ssh`가 왜 원격 서버에 접속하는지
- `docker`가 왜 컨테이너를 별도의 환경처럼 보이게 하는지
- `slurm`이 왜 작업을 제출하고 관리하는 구조인지

즉, 셸과 커널의 관계를 이해하면 리눅스 명령어를 단순 암기보다 구조적으로 이해할 수 있다.

## 요약

1. 응용 프로그램은 사용자가 직접 쓰는 프로그램이다.
2. 셸은 사용자의 명령을 해석해서 커널에 전달한다.
3. 커널은 메모리, 파일, 프로세스, 장치 같은 실제 자원을 관리한다.
4. 리눅스 실습 대부분은 이 세 계층의 관계 위에서 동작한다.
