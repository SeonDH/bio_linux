# [번외] 컴퓨터의 구성 요소와 동작 원리

컴퓨터는 눈에 보이는 화면 하나로만 동작하는 것처럼 보이지만, 실제로는 여러 하드웨어 부품이 순서대로 협력해서 일을 처리한다.  
리눅스를 배울 때는 특히 `CPU`, `메모리`, `저장 장치`, `입출력 장치`가 어떤 역할을 하고, 서로 어떻게 연결되는지 이해하는 것이 중요하다.

## 1. 컴퓨터의 기본 구성 요소

### CPU (중앙처리장치)

CPU는 컴퓨터의 핵심 처리 장치다.  
명령을 읽고, 계산하고, 판단하고, 다음 행동을 결정하는 역할을 한다.

CPU가 하는 일은 단순히 계산만이 아니다.

- 프로그램이 어떤 순서로 실행될지 판단한다.
- 조건문과 반복문을 처리한다.
- 메모리와 저장 장치 사이의 작업 흐름을 조정한다.
- 다른 장치와의 작업 요청을 관리한다.

즉, CPU는 컴퓨터의 "일꾼"이자 "지휘자"에 가깝다.

### 메모리 (RAM)

메모리는 데이터를 임시로 빠르게 저장하는 공간이다.  
CPU가 바로바로 꺼내 쓸 수 있어 속도가 매우 빠르다.

메모리에는 다음과 같은 내용이 잠시 올라간다.

- 실행 중인 프로그램의 코드
- 현재 작업 중인 데이터
- 중간 계산 결과
- 파일을 읽고 처리하는 동안 필요한 정보

메모리는 전원이 꺼지면 내용이 사라진다.  
그래서 오랜 시간 보관해야 하는 데이터는 저장 장치에 두고, 지금 당장 필요한 데이터만 메모리에 올려서 쓴다.

### 저장 장치

저장 장치는 데이터를 오래 보관하는 공간이다.  
HDD와 SSD가 여기에 해당한다.

- **HDD**: 자기 디스크를 이용하는 전통적인 저장 장치
- **SSD**: 반도체 기반 저장 장치로 HDD보다 빠름

저장 장치는 속도는 메모리보다 느리지만, 컴퓨터를 껐다 켜도 데이터가 남는다.  
그래서 문서, 프로그램, 이미지, 분석 결과 파일처럼 오래 남겨야 하는 데이터는 저장 장치에 저장한다.

### 디스크 I/O

디스크 I/O는 저장 장치와 데이터를 주고받는 동작이다.  
여기서 `I/O`는 Input/Output의 약자이므로, 디스크에서 데이터를 읽는 것과 쓰는 것을 모두 포함한다.

예를 들어 다음 작업이 디스크 I/O에 해당한다.

- 파일을 여는 것
- 큰 텍스트 파일을 읽는 것
- 분석 결과를 저장하는 것
- 로그 파일에 계속 기록하는 것
- 여러 프로그램이 동시에 같은 저장 장치에 접근하는 것

디스크 I/O가 많아지면 프로그램이 CPU를 쓰고 싶어도 저장 장치를 기다려야 한다.  
그래서 실제로는 CPU 사용률이 높지 않은데도 전체 작업이 느려질 수 있다.

### 입출력 장치

입출력 장치는 사용자와 컴퓨터를 이어 주는 장치다.

- 입력 장치: 키보드, 마우스, 스캐너 등
- 출력 장치: 모니터, 프린터, 스피커 등

리눅스 실습에서는 키보드와 모니터, 그리고 터미널이 가장 자주 등장한다.  
사용자는 키보드로 명령을 입력하고, 화면으로 결과를 확인한다.

## 2. 어떻게 동작하는가

컴퓨터는 보통 다음 순서로 동작한다.

1. 사용자가 키보드나 프로그램을 통해 작업을 요청한다.
2. CPU가 그 요청을 받아 처리할 준비를 한다.
3. 필요한 데이터가 메모리에 올라간다.
4. CPU가 메모리의 데이터를 읽어 계산하거나 판단한다.
5. 파일을 저장해야 하면 저장 장치에 기록한다.
6. 화면에 보여 줄 내용은 출력 장치를 통해 사용자에게 전달한다.

예를 들어 `ls` 명령을 입력했다고 해보자.

- 사용자는 터미널에 `ls`를 입력한다.
- 셸이 이 명령을 해석한다.
- 커널이 파일 시스템에 접근해 디렉터리 내용을 읽는다.
- CPU는 그 과정을 처리하고, 메모리는 필요한 정보를 잠시 보관한다.
- 결과가 화면에 출력된다.

즉, 사용자가 보는 한 줄의 결과 뒤에는 CPU, 메모리, 저장 장치, 입출력 장치가 모두 함께 움직인다.

## 3. 메모리와 저장 장치가 분리되는 이유

메모리는 빠르지만 휘발성이 있고, 저장 장치는 느리지만 오래 남는다.  
그래서 컴퓨터는 이 두 가지를 분리해서 사용한다.

### 메모리가 필요한 이유

- 프로그램을 빠르게 실행하려면 자주 쓰는 데이터를 가까이에 둬야 한다.
- CPU는 매우 빠르기 때문에, 느린 저장 장치에서 매번 직접 읽으면 기다리는 시간이 길어진다.
- 그래서 실행 중인 데이터는 메모리에 올려 두고 CPU가 바로 처리한다.

### 저장 장치가 필요한 이유

- 종료 후에도 남아 있어야 하는 데이터가 필요하다.
- 문서, 사진, 분석 결과, 프로그램 파일을 오래 보관해야 한다.
- 메모리만으로는 전원을 끄면 모두 사라지므로 저장 장치가 필요하다.

## 4. 왜 느려질 수 있는가

컴퓨터가 느려지는 이유는 여러 가지가 있다.

- CPU가 너무 바쁘면 계산이 밀린다.
- 메모리가 부족하면 더 느린 저장 장치를 대신 사용하게 된다.
- 저장 장치가 느리면 파일 읽기와 쓰기가 병목이 된다.
- 입출력이 많으면 프로그램이 CPU보다 I/O를 기다리는 시간이 길어진다.
- 디스크 I/O가 몰리면 프로세스가 대기 상태에 오래 머문다.

리눅스 실습에서 자주 보는 현상도 대부분 이 구조와 연결된다.

- 메모리 부족으로 swap이 생기면 디스크 사용량이 올라간다.
- 큰 파일을 많이 읽는 작업은 CPU보다 디스크 I/O가 병목이 된다.
- 동시에 여러 작업을 하면 CPU, 메모리, 디스크가 서로 경쟁할 수 있다.
- 디스크 I/O가 몰리면 `D` 상태처럼 중단하기 어려운 대기 상태가 나타날 수 있다.

## 5. 리눅스 실습과의 연결

이 구조를 알아 두면 리눅스에서 다음 개념을 이해하기 쉬워진다.

- `ps`, `top`, `htop`: CPU와 메모리 사용 상태 확인
- `df`, `du`: 저장 장치와 디스크 사용량 확인
- `chmod`, `chown`: 파일 접근 방식 이해
- `scp`, `rsync`: 네트워크를 통해 파일을 옮길 때의 흐름 이해
- `docker`: 프로세스와 파일 시스템을 분리해 실행 환경을 만드는 개념 이해
- `slurm`: CPU와 메모리 자원을 나눠 쓰는 작업 관리 개념 이해

즉, 컴퓨터의 구성 요소를 이해하면 이후의 실습이 단순한 명령어 암기가 아니라 구조 이해로 바뀐다.

## 요약

1. CPU는 계산과 판단을 담당한다.
2. 메모리는 실행 중인 데이터를 빠르게 임시 저장한다.
3. 저장 장치는 데이터를 오래 보관한다.
4. 디스크 I/O는 저장 장치와 데이터를 주고받는 동작이다.
5. 입출력 장치는 사용자와 컴퓨터를 연결한다.
6. 리눅스 실습은 이 다섯 가지가 어떻게 함께 움직이는지 이해하면 훨씬 쉬워진다.
