# [번외] 네트워크 용어

> 이 문서는 네트워크 전공 수준의 이론 정리가 아니라, 리눅스 서버 접속과 파일 전송 실습 중 자주 만나는 용어를 빠르게 확인하기 위한 참고 자료이다.

## 꼭 알아두면 좋은 용어

| 용어 | 의미 | 실습에서 만나는 상황 |
| --- | --- | --- |
| IP 주소 | 컴퓨터나 서버를 찾기 위한 주소 | `ssh user@192.168.0.10` |
| 도메인 | 사람이 읽기 쉬운 서버 이름 | `koreabio.limeops.co.kr` |
| DNS | 도메인을 IP 주소로 바꿔주는 시스템 | 도메인으로 접속했는데 주소를 찾지 못할 때 |
| 포트 | 한 서버 안에서 서비스를 구분하는 번호 | SSH 기본 포트 `22`, 실습 서버 포트 `1119` |
| 프로토콜 | 통신 방식에 대한 약속 | SSH, HTTP, HTTPS, TCP, UDP |
| SSH | 원격 서버에 안전하게 접속하는 방법 | `ssh -p 1119 user@server` |
| SCP | SSH를 이용해 파일을 복사하는 방법 | 로컬 파일을 서버로 올릴 때 |
| rsync | 변경된 파일 중심으로 효율적으로 동기화하는 도구 | 여러 파일을 반복해서 서버에 올릴 때 |
| 방화벽 | 허용된 IP/포트만 통과시키는 보안 장치 | 포트가 막혀 접속이 안 될 때 |
| NAT | 사설 IP가 외부 인터넷과 통신할 수 있게 주소를 바꾸는 방식 | 집/회사 공유기 뒤에서 인터넷을 사용할 때 |
| 사설 IP | 내부 네트워크에서만 쓰는 IP | `192.168.x.x`, `10.x.x.x` |
| 공인 IP | 인터넷에서 직접 식별 가능한 IP | 외부에서 서버에 접속할 때 |

## IP 주소와 도메인

서버에 접속하려면 서버의 위치를 알아야 한다. 이때 사용하는 값이 IP 주소 또는 도메인이다.

```bash
ssh user@192.168.0.10
ssh user@koreabio.limeops.co.kr
```

- `192.168.0.10`: IP 주소
- `koreabio.limeops.co.kr`: 도메인

도메인을 사용하면 DNS가 내부적으로 IP 주소를 찾아준다.

```bash
ping koreabio.limeops.co.kr
```

도메인 접속이 안 될 때는 다음 두 가지를 구분해서 생각한다.

- 서버 자체가 응답하지 않는 경우
- 도메인을 IP 주소로 바꾸는 DNS 조회가 실패한 경우

## 포트

포트는 같은 서버 안에서 어떤 서비스로 접속할지 구분하는 번호이다.

```bash
ssh -p 1119 user@koreabio.limeops.co.kr
```

위 명령에서:

- `koreabio.limeops.co.kr`: 접속할 서버
- `1119`: 접속할 포트
- `user`: 로그인할 사용자 계정

자주 보는 포트 예시는 다음과 같다.

| 포트 | 용도 |
| --- | --- |
| 22 | SSH 기본 포트 |
| 80 | HTTP |
| 443 | HTTPS |

> `ssh`에서는 포트 옵션이 소문자 `-p`이고, `scp`에서는 대문자 `-P`이다.

## TCP와 UDP

TCP와 UDP는 데이터를 주고받는 대표적인 전송 방식이다.

| 구분 | 특징 | 예 |
| --- | --- | --- |
| TCP | 연결을 확인하며 안정적으로 전송 | SSH, HTTP, HTTPS |
| UDP | 연결 확인 절차가 적어 빠르지만 손실 가능 | DNS 조회, 일부 스트리밍 |

리눅스 실습에서는 대부분 TCP 기반 명령어를 사용한다고 보면 된다.

## 방화벽

방화벽은 특정 IP나 포트의 접속을 허용하거나 차단한다.

예를 들어 서버가 정상이고 계정도 맞는데 접속이 안 된다면 다음을 확인한다.

```bash
nc -vz koreabio.limeops.co.kr 1119
```

- `succeeded` 또는 `open`과 비슷한 메시지: 해당 포트 접속 가능
- `timed out`, `refused` 등: 네트워크, 방화벽, 서버 설정 확인 필요

## 사설 IP, 공인 IP, NAT

집이나 회사에서 자주 보는 `192.168.x.x`, `10.x.x.x` 같은 주소는 사설 IP이다. 사설 IP는 내부 네트워크에서만 사용된다.

대표적인 사설 IP 대역:

| 대역 | 예 |
| --- | --- |
| `10.0.0.0/8` | `10.1.2.3` |
| `172.16.0.0/12` | `172.16.5.10` |
| `192.168.0.0/16` | `192.168.0.10` |

사설 IP를 사용하는 PC가 인터넷에 접속할 수 있는 이유는 공유기나 네트워크 장비가 NAT를 통해 공인 IP로 변환해주기 때문이다.

## localhost와 127.0.0.1

`localhost`와 `127.0.0.1`은 현재 자기 자신의 컴퓨터를 의미한다.

```bash
curl http://localhost:8000
curl http://127.0.0.1:8000
```

도커나 웹 실습에서 `localhost:8000`처럼 보이면 “내 PC의 8000번 포트로 접속한다”는 뜻이다.

## 네트워크 확인 명령어

| 명령어 | 용도 |
| --- | --- |
| `ip addr show` | 내 IP 주소 확인 |
| `ping 주소` | 대상 서버가 응답하는지 확인 |
| `ss -tuln` | 현재 열려 있는 포트 확인 |
| `nc -vz 호스트 포트` | 특정 포트 접속 가능 여부 확인 |
| `tracepath 주소` | 목적지까지의 네트워크 경로 확인 |

예:

```bash
ip addr show
ping -c 4 8.8.8.8
ss -tuln
nc -vz koreabio.limeops.co.kr 1119
tracepath 8.8.8.8
```

## 알아만 두면 되는 용어

아래 용어들은 네트워크 전공 수업에서는 중요하지만, 이 과정에서는 깊게 다루지 않아도 된다.

| 용어 | 간단한 의미 |
| --- | --- |
| OSI 7계층 | 네트워크 통신 구조를 7단계로 나누어 설명하는 모델 |
| 라우팅 | 데이터가 목적지까지 가는 경로를 정하는 과정 |
| 서브넷 | 하나의 네트워크를 더 작은 네트워크로 나눈 것 |
| VLAN | 물리적으로 같은 네트워크를 논리적으로 분리하는 기술 |
| VPN | 외부에서 내부망에 안전하게 접속하기 위한 가상 사설망 |
| DHCP | IP 주소를 자동으로 할당하는 방식 |
| MAC 주소 | 네트워크 장비에 붙는 하드웨어 주소 |

## 정리

이 과정에서는 네트워크를 깊게 계산하거나 설계할 필요는 없다. 다음 정도만 이해하면 서버 접속, 파일 전송, Docker/SLURM 실습을 따라가는 데 충분하다.

1. 서버는 IP 주소나 도메인으로 찾는다.
2. 포트는 서버 안의 서비스 번호이다.
3. SSH와 SCP는 원격 서버 접속과 파일 전송에 사용한다.
4. 접속이 안 될 때는 주소, 포트, 계정, 방화벽을 차례로 확인한다.
5. `ip addr`, `ping`, `ss`, `nc` 정도의 확인 명령어를 알아두면 된다.
